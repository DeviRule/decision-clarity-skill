# Output Patterns

This is the single source of truth for response structure. SKILL.md modes and workflow outputs should align to these patterns.

Use these patterns when a structured answer improves clarity.

## Pattern 1: Quick reframe

**Maps to:** Mode A. Use for short, single-factor questions.

- Real issue
- Hidden assumption
- Main constraint
- Simpler conclusion
- Next move

## Pattern 2: Structured decision analysis

**Maps to:** Mode B. Use for multi-factor decisions with real stakes.

- Real goal
- Assumptions
- Basic facts
- Hard constraints
- Soft constraints
- Simpler options
- Recommendation
- First test or next step

## Pattern 3: Reasoning audit

**Maps to:** Mode C. Use when evaluating the user's own reasoning.

- Real question
- Weak assumptions
- Missing evidence or contradiction
- Simpler interpretation
- Refined judgment
- Next checkpoint

## Pattern 4: Option comparison

**Use when:** The user presents 2+ named options for head-to-head evaluation.

- Real decision (what is actually being chosen)
- Decision criteria (what matters most)
- Per-option assessment (each option against each criterion)
- Winner and why
- What would flip the ranking
- Next move

## Pattern 5: Workflow simplification

**Use when:** The problem is an overbuilt process, system, or workflow.

- Desired outcome
- Existing complexity
- What is essential
- What can be removed (and why removal is safe)
- Simplified design
- First removal or test
