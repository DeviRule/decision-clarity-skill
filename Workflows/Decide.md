# Decide Workflow

Use this workflow to convert analysis into a practical judgment or next move.

## When to use

Use this workflow when:

- the user needs a recommendation
- multiple options remain
- the issue is clearer but still unresolved
- analysis must become action
- the conversation risks ending in abstract reflection

## Standalone entry

If prior workflows (Clarify, Deconstruct, Simplify) have not run, first identify:
- the real goal (what outcome matters)
- the key facts and hard constraints
- the surviving options

Then proceed to Step 1. If you cannot identify these, run Clarify or Deconstruct first.

## Core question

Given the clarified problem, real facts, and simplified options, what is the best current move?

## Process

### Step 1: Keep only what survived scrutiny

Retain only:

- the real goal
- supported facts
- hard constraints
- lower-assumption options
- relevant tradeoffs

### Step 2: Remove what did not survive

Discard:

- vague framing
- unsupported assumptions
- inherited but challengeable defaults
- complexity without clear value
- weak options that only look complete

### Step 3: Define decision criteria

Before choosing, name what "best" means for this decision:
- What is being optimized for? (speed, cost, learning, risk reduction, value)
- What is acceptable to sacrifice?
- What weight does reversibility carry?

### Step 4: Compare surviving options

If multiple options remain:
- Compare on the criteria from Step 3
- Note which option wins on which criteria
- Identify where options are equivalent (making the choice low-stakes on that dimension)

If only one option remains, skip to Step 5.

### Step 5: State the best current judgment

Produce one of:

- a recommendation
- a priority order
- a preferred option
- a decision rule
- a cleaner framing of the remaining choice

### Step 6: Name the key tradeoffs

Be explicit about what is being sacrificed, accepted, or still uncertain. Rank tradeoffs by consequence. State which tradeoff the recommendation is optimizing for.

### Step 7: Pre-mortem

Assume the recommendation was implemented and failed. Ask:
- What went wrong?
- Which assumption was false?
- What was not anticipated?

If a plausible, high-consequence failure path emerges, either add a mitigation to the recommendation or reconsider it. If no serious failure path is apparent, proceed with higher confidence.

### Step 8: Check the outside view

Ask: how do similar decisions turn out for similar people in similar situations? What is the base rate?

This is one sentence, not a research project. Example: "Most first-time founders fail within 2 years, so the recommendation should include a 'what if this fails' fallback."

If no relevant base rate exists, note that and proceed.

### Step 9: Assess reversibility

Is the recommended action reversible?
- **If yes:** bias toward acting now and learning.
- **If no:** look for a reversible version first (pilot, prototype, limited test). Only recommend the irreversible version when no reversible alternative exists or when delay costs exceed reversal costs.

### Step 10: Check confidence

Is the available information sufficient to commit?
- **If yes:** proceed to next move.
- **If no:** the best recommendation may be "do not decide yet — gather X specific information first." This is a legitimate output, not a failure. Name exactly what information would change the recommendation.

### Step 11: End with an immediate next move

Give:

- a first action
- a smallest useful experiment
- a validation step
- a question that must be answered next

## Output format

- What is now clear
- What still matters
- Decision criteria
- Best current recommendation
- Key tradeoffs (ranked by consequence)
- Pre-mortem: how this could fail
- Reversibility assessment
- Next move

## Quality bar

A good decision output should:
- be more decisive and grounded than the original discussion
- name what it is optimizing for and what it is sacrificing
- assess whether the action is reversible
- end with a concrete, executable next move
- name at least one plausible way the recommendation could fail
